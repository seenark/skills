# Common README Mistakes

## Mistake: README สั้นจนไม่บอก purpose

- Symptom: คนอ่านถามว่า project นี้ใช้ทำอะไร
- Why: ผู้เขียนคิดว่าชื่อ project เพียงพอ
- Debug: ให้คนใหม่อ่าน README แล้วถามว่าเขาอธิบาย project กลับได้ไหม
- Safe fix: เพิ่ม Purpose, What you will learn, Next step
- Prevention: เขียน README โดยคิดถึงคนที่ไม่รู้ context มาก่อน
